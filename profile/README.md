## About

**Collaborative Software** is not a company, nor an official university department. It is an independent organization created by Computer Science and Software Development students to centralize the projects we build together throughout our degree.

Our goal is simple: bring every group project, academic assignment, and side experiment into one organized space that reflects how we work, how we grow, and how we collaborate as a team. This organization serves as a collective portfolio — a place where recruiters, professors, colleagues, and fellow developers can see not only what we build, but how we build it.

Every repository here represents teamwork: shared decisions, code reviews, discussions, and iteration. We treat these academic projects with the same care and discipline we'd bring to professional software development.

## Our Philosophy

- **Continuous learning** — every project is an opportunity to learn something new.
- **Collaborative development** — we build together, using pull requests, code reviews, and shared decision-making.
- **Clean code** — we aim for readable, maintainable, and well-structured code.
- **Knowledge sharing** — what one of us learns, all of us benefit from.
- **Proper version control** — meaningful commits, clear branches, and organized history.
- **Software Engineering best practices** — documentation, planning, and process matter as much as the code itself.

---

## Repository Structure

Each repository in this organization is treated as an independent project. That means:

- Its own dedicated `README.md`, explaining the project's purpose, setup, and usage.
- Its own development history, reflecting the team's actual workflow and decisions.
- Its own documentation, adapted to the technologies and scope of that specific project.

We believe this structure keeps every project understandable on its own, without requiring context from the organization as a whole.

---

## Contribution

All projects here are developed and maintained by members of the organization as part of academic coursework, extension projects, hackathons, or personal initiatives related to software development.

Contribution guidelines are specific to each repository — check the individual project's `README.md` and contribution notes before proposing changes. In general, we welcome discussion, feedback, and collaboration from anyone interested in how we build our projects.

## Final Note

Every project here is a step in our journey as developers — built together, refined through collaboration, and shaped by continuous learning. We believe that good software isn't just written; it's built as a team.
